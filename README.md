    <article class="markdown-body entry-content container-lg" itemprop="text"><h1><a id="user-content-metarmap" class="anchor" aria-hidden="true" href="#metarmap"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>METARMap</h1>
<p>Raspberry Pi project to visualize flight conditions on a map using WS8211 LEDs addressed via NeoPixel</p>
<h2><a id="user-content-detailed-instructions" class="anchor" aria-hidden="true" href="#detailed-instructions"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Detailed instructions</h2>
<p>I've created detailed instructions about the setup and parts used here: <a href="https://slingtsi.rueker.com/making-a-led-powered-metar-map-for-your-wall/" rel="nofollow">https://slingtsi.rueker.com/making-a-led-powered-metar-map-for-your-wall/</a></p>
<h2><a id="user-content-software-setup" class="anchor" aria-hidden="true" href="#software-setup"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Software Setup</h2>
<ul>
<li>Install <a href="https://www.raspberrypi.org/software/" rel="nofollow">Raspberry Pi OS Lite</a> on SD card</li>
<li><a href="https://medium.com/@danidudas/install-raspbian-jessie-lite-and-setup-wi-fi-without-access-to-command-line-or-using-the-network-97f065af722e" rel="nofollow">Enable Wi-Fi and SSH</a></li>
<li>Install SD card and power up Raspberry Pi</li>
<li>SSH (using <a href="https://www.putty.org" rel="nofollow">Putty</a> or some other SSH tool) into the Raspberry and configure password and timezones
<ul>
<li><code>passwd</code></li>
<li><code>sudo raspi-config</code></li>
</ul>
</li>
<li>Update packages
<ul>
<li><code>sudo apt-get update</code></li>
<li><code>sudo apt-get upgrade</code></li>
</ul>
</li>
<li>Copy the <strong><a href="/prueker/METARMap/blob/main/metar.py">metar.py</a></strong>, <strong><a href="/prueker/METARMap/blob/main/pixelsoff.py">pixelsoff.py</a></strong>, <strong><a href="/prueker/METARMap/blob/main/airports">airports</a></strong>, <strong><a href="/prueker/METARMap/blob/main/refresh.sh">refresh.sh</a></strong> and <strong><a href="/prueker/METARMap/blob/main/lightsoff.sh">lightsoff.sh</a></strong> scripts into the pi home directory (/home/pi)</li>
<li>Install python3 and pip3 if not already installed
<ul>
<li><code>sudo apt-get install python3</code></li>
<li><code>sudo apt-get install python3-pip</code></li>
</ul>
</li>
<li>Install required python libraries for the project
<ul>
<li><a href="https://learn.adafruit.com/neopixels-on-raspberry-pi/python-usage" rel="nofollow">Neopixel</a>: <code>sudo pip3 install rpi_ws281x adafruit-circuitpython-neopixel</code></li>
</ul>
</li>
<li>Attach WS8211 LEDs to Raspberry Pi, if you are using just a few, you can connect the directly, otherwise you may need to also attach external power to the LEDs. For my purpose with 22 powered LEDs it was fine to just connect it directly. You can find <a href="https://learn.adafruit.com/neopixels-on-raspberry-pi/raspberry-pi-wiring" rel="nofollow">more details about wiring here</a>.</li>
<li>Test the script by running it directly (it needs to run with root permissions to access the GPIO pins):
<ul>
<li><code>sudo python3 metar.py</code></li>
</ul>
</li>
<li>Make appropriate changes to the <strong><a href="/prueker/METARMap/blob/main/airports">airports</a></strong> file for the airports you want to use and change the <strong><a href="/prueker/METARMap/blob/main/metar.py">metar.py</a></strong> and <strong><a href="/prueker/METARMap/blob/main/pixelsoff.py">pixelsoff.py</a></strong> script to the correct <strong><code>LED_COUNT</code></strong> (including NULLs if you have LEDS in between airports that will stay off) and <strong><code>LED_BRIGHTNESS</code></strong> if you want to change it</li>
<li>To run the script automatically when you power the Raspberry Pi, you will need to grant permissions to execute the <strong><a href="/prueker/METARMap/blob/main/refresh.sh">refresh.sh</a></strong> and <strong><a href="/prueker/METARMap/blob/main/lightsoff.sh">lightsoff.sh</a></strong> script and read permissions to the <strong><a href="/prueker/METARMap/blob/main/airports">airports</a></strong>, <strong><a href="/prueker/METARMap/blob/main/metar.py">metar.py</a></strong> and <strong><a href="/prueker/METARMap/blob/main/pixelsoff.py">pixelsoff.py</a></strong> script using chmod:
<ul>
<li><code>chmod +x filename</code> will grant execute permissions</li>
<li><code>chmod +r filename</code> will grant write permissions</li>
</ul>
</li>
<li>To have the script start up automatically and refresh in regular intervals, use crontab and set the appropriate interval. For an example you can refer to the <a href="/prueker/METARMap/blob/main/crontab">crontab</a> file in the GitHub repo (make sure you grant the file execute permissions beforehand to the refresh.sh and lightsoff.sh file). To edit your crontab type: <strong><code>crontab -e</code></strong>, after you are done with the edits, exit out by pressing <strong>ctrl+x</strong> and confirm the write operation
<ul>
<li>The sample crontab will run the script every 5 minutes (the */5) between the hours of 7 to 21, which includes the 21 hour, so it means it will run until 21:55</li>
<li>Then at 22:05 it will run the lightsoff.sh script, which will turn all the lights off</li>
</ul>
</li>
</ul>
<h2><a id="user-content-additional-wind-condition-blinkingfading-functionality" class="anchor" aria-hidden="true" href="#additional-wind-condition-blinkingfading-functionality"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Additional Wind condition blinking/fading functionality</h2>
<p>I recently expanded the script to also take wind condition into account and if the wind exceeds a certain threshold, or if it is gusting, make the LED for that airport either blink on/off or to fade between  two shades of the current flight category color.</p>
<p>If you want to use this extra functionality, then inside the <strong><a href="/prueker/METARMap/blob/main/metar.py">metar.py</a></strong> file set the <strong><code>ACTIVATE_WINDCONDITION_ANIMATION</code></strong> parameter to <strong>True</strong>.</p>
<ul>
<li>There are a few additional parameters in the script you can configure to your liking:
<ul>
<li><code>FADE_INSTEAD_OF_BLINK</code> - set this to either <strong>True</strong> or <strong>False</strong> to switch between fading or blinking for the LEDs when conditions are windy</li>
<li><code>WIND_BLINK_THRESHOLD</code> - in Knots for normal wind speeds currently at the airport</li>
<li><code>ALWAYS_BLINK_FOR_GUSTS</code> - If you always want the blinking/fading to happen for gusts, regardless of the wind speed</li>
<li><code>BLINKS_SPEED</code> - How fast the blinking happens, I found 1 second to be a happy medium so it's not too busy, but you can also make it faster, for example every half a second by using 0.5</li>
<li><code>BLINK_TOTALTIME_SECONDS</code> = How long do you want the script to run. I have this set to 300 seconds as I have my crontab setup to re-run the script every 5 minutes to get the latest weather information</li>
<li><code>HIGH_WINDS_THRESHOLD</code> - If you want LEDs to flash to Yellow for particularly high winds beyond the normal <code>WIND_BLINK_THRESHOLD</code> then set this variable in knots. If you only want normal blinking/fading based on <code>WIND_BLINK_THRESHOLD</code> then set the value for <code>HIGH_WINDS_THRESHOLD</code> to <strong><code>-1</code></strong></li>
</ul>
</li>
</ul>
<h2><a id="user-content-additional-lightning-in-the-vicinity-blinking-functionality" class="anchor" aria-hidden="true" href="#additional-lightning-in-the-vicinity-blinking-functionality"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Additional Lightning in the vicinity blinking functionality</h2>
<p>After the recent addition for wind condition animation, I got another request from someone if I could add a white blinking animation to represent lightning in the area.
Please note that due to the nature of the METAR system, this means that the METAR for this airport reports that there is Lightning somewhere in the vicinity of the airport, but not necessarily right at the airport.</p>
<p>If you want to use this extra functionality, then inside the <strong><a href="/prueker/METARMap/blob/main/metar.py">metar.py</a></strong> file set the <strong><code>ACTIVATE_LIGHTNING_ANIMATION</code></strong> parameter to <strong>True</strong>.</p>
<ul>
<li>This shares two configuration parameters together with the wind animation that you can modify as you like:
<ul>
<li><code>BLINKS_SPEED</code> - How fast the blinking happens, I found 1 second to be a happy medium so it's not too busy, but you can also make it faster, for example every half a second by using 0.5</li>
<li><code>BLINK_TOTALTIME_SECONDS</code> = How long do you want the script to run. I have this set to 300 seconds as I have my crontab setup to re-run the script every 5 minutes to get the latest weather information</li>
</ul>
</li>
</ul>
<h2><a id="user-content-additional-led-dimming-functionality-based-on-time-of-day" class="anchor" aria-hidden="true" href="#additional-led-dimming-functionality-based-on-time-of-day"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Additional LED dimming functionality based on time of day</h2>
<p>This optional functionality allows you to run the LEDs at a dimmed lower level between a certain time of the day.</p>
<p>If you want to use this extra functionality, then inside the <strong><a href="/prueker/METARMap/blob/main/metar.py">metar.py</a></strong> file set the <strong><code>ACTIVATE_DAYTIME_DIMMING</code></strong> parameter to <strong>True</strong>.
Set the <code>LED_BRIGHTNESS_DIM</code> setting to the level you want to run when dimmed.</p>
<p>For time timings of the dimming there are two options:</p>
<ul>
<li>Fixed time of day dimming:
<ul>
<li><code>BRIGHT_TIME_START</code> - Set this to the beginning of the day when you want to run at the normal <code>LED_BRIGHTNESS</code> level</li>
<li><code>DIM_TIME_START</code> - Set this to the time where you want to run at a different <code>LED_BRIGHTNESS_DIM</code> level</li>
</ul>
</li>
<li>Dimming based on local sunrise/sunset:
<ul>
<li>For this to work, you need to install an additional library, run:
<ul>
<li><code>sudo pip3 install astral</code></li>
</ul>
</li>
<li><code>USE_SUNRISE_SUNSET</code> - Set this to <strong>True</strong> to use the dimming based on sunrise and sunset</li>
<li><code>LOCATION</code> - set this to the city you want to use for sunset/sunrise timings
<ul>
<li>Use the closest city from the list of supported cities from <a href="https://astral.readthedocs.io/en/latest/#cities" rel="nofollow">https://astral.readthedocs.io/en/latest/#cities</a></li>
</ul>
</li>
</ul>
</li>
</ul>
<h2><a id="user-content-additional-mini-display-to-show-metar-information-functionality" class="anchor" aria-hidden="true" href="#additional-mini-display-to-show-metar-information-functionality"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Additional mini display to show METAR information functionality</h2>
<p>This optional functionality allows you to connect a small mini LED display to show the METAR information of the airports.</p>
<p>For this functionality to work, you will need to buy a compatible LED display and enable and install a few additional things.</p>
<p>I've written up some details on the display I used and the wiring here: <a href="https://slingtsi.rueker.com/adding-a-mini-display-to-show-metar-information-to-the-metar-map/" rel="nofollow">https://slingtsi.rueker.com/adding-a-mini-display-to-show-metar-information-to-the-metar-map/</a></p>
<p>To support the display you need to enable a few new libraries and settings on the raspberry pi.</p>
<ul>
<li><a href="https://learn.adafruit.com/adafruits-raspberry-pi-lesson-4-gpio-setup/configuring-i2c" rel="nofollow">Enable I2C</a></li>
<li><code>sudo raspi-config</code></li>
<li>Interface Options</li>
<li>I2C</li>
<li>reboot the Reboot the Raspberry Pi <code>sudo reboot</code></li>
<li>Verify your wiring is working and I2C is enabled
<ul>
<li><code>sudo apt-get install i2c-tools</code></li>
<li><code>sudo i2cdetect -y 1</code> - this should show something connected at <strong>3C</strong></li>
</ul>
</li>
<li>install python library for the display
<ul>
<li><code>sudo pip3 install adafruit-circuitpython-ssd1306</code></li>
<li><code>sudo pip3 install pillow</code></li>
</ul>
</li>
<li>install additional libraries needed to fill the display
<ul>
<li><code>sudo apt-get install ttf-dejavu</code></li>
<li><code>sudo apt-get install libjpeg-dev -y</code></li>
<li><code>sudo apt-get install zlib1g-dev -y</code></li>
<li><code>sudo apt-get install libfreetype6-dev -y</code></li>
<li><code>sudo apt-get install liblcms1-dev -y</code></li>
<li><code>sudo apt-get install libopenjp2-7 -y</code></li>
<li><code>sudo apt-get install libtiff5 -y</code></li>
</ul>
</li>
<li>copy new file <strong><a href="/prueker/METARMap/blob/main/displaymetar.py">displaymetar.py</a></strong> into the same folder as <strong><a href="/prueker/METARMap/blob/main/metar.py">metar.py</a></strong></li>
<li>Use the latest version of <strong><a href="/prueker/METARMap/blob/main/metar.py">metar.py</a></strong> and <strong><a href="/prueker/METARMap/blob/main/pixelsoff.py">pixelsoff.py</a></strong> for the new functionality</li>
<li>Configure <strong><a href="/prueker/METARMap/blob/main/metar.py">metar.py</a></strong> and set <strong><code>ACTIVATE_EXTERNAL_METAR_DISPLAY</code></strong> parameter to <strong>True</strong>.</li>
<li>Configure the <code>DISPLAY_ROTATION_SPEED</code> to your desired timing, I'm using 5 seconds for mine.</li>
<li>If you want to only show a subset of the airports on the display, create a new file in the folder called <strong>displayairports</strong> and add the airports that you want to be shown on the display to it</li>
</ul>
<h2><a id="user-content-legend" class="anchor" aria-hidden="true" href="#legend"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Legend</h2>
<p>If you want an interactive Legend to illustrate the possible behaviors you can do so by adding an additional up to 7 LEDs after the last LED based on your number of LEDs of the airports in the <strong>airports</strong> file</p>
<ul>
<li>Set <code>SHOW_LEGEND</code> to <strong>True</strong> to use this feature</li>
<li>If you want to skip some LEDs after your last airport before the legend, you can set <code>OFFSET_LEGEND_BY</code> to the number of LEDs to skip</li>
<li><strong>Note</strong>: The Lightning and Wind Condition LEDs will only show if you are actually using these features based on the <code>ACTIVATE_LIGHTNING_ANIMATION</code>, <code>ACTIVATE_WINDCONDITION_ANIMATION</code> and <code>HIGH_WINDS_THRESHOLD</code> variables.
<ul>
<li>If you are not using any of these, then you only need 4 LEDs for the basic flight conditions for the Legend</li>
<li>If you are only using the Wind condition feature, but not the Lightning, you will still need the total of 7 LEDs (but the 5th LED for Lightning will just stay blank) or you'd have to change the order in the code</li>
</ul>
</li>
</ul>
